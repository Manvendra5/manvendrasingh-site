JavaScript: Variables 101

September 26, 2021

Creating variables and using them can be thought of as one of the earliest and simplest concepts when we start learning to program. In JavaScript, if you don't have a good understanding of variables it can create a lot of problems.

What are Variables?

Variables are containers for storing data (values). JavaScript variables are containers for storing data values. There are some rules that one should keep in mind before creating variables:

  • Variable names can contain letters, digits, underscores, and dollar signs.
  • Variable names should start with a letter.
  • Variable names are case sensitive for eg read and READ are two different variable.
  • Some names are reserved in JavaScript. So, we cannot use the same keywords for naming variables.

Assignment Operator

In JavaScript, the equal sign (=) is known as "assignment" operator, not an "equal to" operator. Remember equality operators like '==' are different from assignment operator.

There are three ways to create a variable in JavaScript:

  • var
  • let
  • const

Creating variables with var, let and const

There are pros and cons of using var, let and const. We will discuss them in the next section.

understanding the difference between var, let and const

The value of variables created using var and let can be updated/changed but const as the name suggest has a constant value that cannot be changed. If you tried changing const variable value it will throw an error. The error describes what we did wrong.

One more thing we need to understand is that creating variables without using var, let and const is a bad practice. By default JavaScript will create a var variable if you do not follow this rule which will further lead to problems and you don't want that. Before ES6, mostly var was used but JavaScript developers understood the cons of using var and thus came up with new ways to create variables.

understanding Scoping in JavaScript

Before we discuss about the second difference we need to know about scoping. Scoping is a big topic and has a lot of importance in JavaScript. So, right now we only need to know how scoping affects variables.

  • var variables are scoped differently than let and const variables.
  • var variables are function scoped variables (they are only available within the parent function)
  • let and const variables ar block scoped variables.

So, what is function scoped? Anything that sits inside { and } is called a block. You must have written code inside curly braces while working with functions, loops and conditional statements.

The scope is global when a var variable is declared outside a function. This means that any variable that is declared with var outside a function block is available for use in the whole window. var is function scoped when it is declared within a function. This means that it is available and can be accessed only within that function.

variable declared in a block with let is only available for use within that block.

Just like let, const is only available for use within a block but unlike let const variables cannot be updated.
